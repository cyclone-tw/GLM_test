# 3D 動畫抽籤系統設計文件

## 概述

一個可自訂名單和獎項的 3D 動畫抽籤 HTML 應用程式，採用球體式 3D 動畫效果，支援多種主題切換和豐富的互動功能。

## 核心功能

### 1. 3D 動畫效果

**球體式抽籤動畫：**
- 名字以 3D 文字或卡片形式漂浮在球體空間中
- 點擊抽獎時，球體加速旋轉
- 旋轉逐漸減速後，中獎者名字飛出並放大顯示
- 使用 Three.js 實現 3D 渲染

**中獎動畫：**
- 煙火特效（使用 canvas 粒子系統）
- 彩帶/紙屑飄落（使用 canvas 粒子系統）
- 中獎者顯示：從 3D 場景切換到 2D overlay，顯示中獎者名字與獎項，使用 CSS 動畫實現發光放大效果

### 2. 獎項與名單管理

**獎項處理邏輯：**
- 獎項與名單一起抽：先設好獎項清單，每次抽獎從名單抽出得獎者並綁定獎項
- 中獎者從名單移除，不重複中獎
- 獎項抽完後可繼續抽剩餘名單（無獎項綁定）
- 獎項抽完時 UI 顯示「獎項: 無（繼續抽名）」，歷史記錄 prize 欄位為 null

**輸入方式：**
- 列表管理介面：新增/刪除/編輯/拖曳排序
- 批次匯入：文字框貼上，一行一項
- Google Sheets 匯入：透過 Google Sheets API（使用者提供 API Key，僅支援已發布為網頁的公開試算表）
- Google 表單匯入：Google 表單的回應會自動儲存到 Google Sheets，因此使用相同的 Google Sheets API 匯入功能。使用者需提供表單回應試算表的網址。

### 3. 主題系統

**預設主題：**
1. 科技感 - 深色背景、霓虹光效、粒子特效
2. 節慶派對 - 彩色繽紛、金幣彩帶、熱鬧歡樂
3. 簡約現代 - 淺色背景、扁平設計、專業感
4. 自訂 - 使用者可調整以下屬性：
   - 背景顏色（色碼選擇器）
   - 主要文字顏色
   - 強調色（用於中獎者名字、按鈕）
   - 粒子/特效顏色

**主題切換：**
- 即時切換，不需重新載入
- 主題設定儲存於 localStorage

### 4. 音效系統

- 滾動音效：抽獎進行時播放
- 中獎音效：揭曉中獎者時播放
- 音效開關：使用者可關閉音效

**音效來源：**
- 使用內嵌 base64 編碼的 MP3 音效檔案（避免外部依賴，MP3 有最佳瀏覽器支援）
- 音效可使用免費音效庫（如 Freesound.org 的 CC0 授權音效）
- 預載音效以避免延遲

### 5. 全螢幕模式

- 投影模式：隱藏設定區，只顯示 3D 動畫區
- 適合會議/活動現場投影使用
- 快速鍵切換（如 F 鍵）

### 6. 歷史記錄

- 記錄每次抽獎結果（時間、獎項、中獎者）
- 可查看歷史記錄列表
- 可清除歷史記錄
- 儲存於 localStorage

**歷史記錄資料結構：**
```javascript
{
  id: string,           // 唯一識別碼 (UUID)
  timestamp: number,    // Unix 時間戳（毫秒）
  prize: string | null, // 獎項名稱，無獎項時為 null
  winner: string        // 中獎者名字
}
```

### 7. 匯出功能

**結果匯出格式：**
- CSV：試算表格式，包含欄位：抽獎序號, 時間, 獎項, 中獎者
- JSON：結構化資料，包含完整歷史記錄陣列
- 圖片：中獎畫面截圖（使用 html2canvas）

**匯出按鈕位置：**
- 位於「歷史記錄」分頁的列表上方
- 提供 [匯出 CSV] [匯出 JSON] 按鈕
- 單次中獎畫面提供 [截圖儲存] 按鈕

### 檔案結構

```
lottery.html          # 單一 HTML 檔案（內含 CSS 和 JS）
```

### 資料儲存

- 名單、獎項：localStorage
- 歷史記錄：localStorage
- 主題設定：localStorage
- Google API Key：localStorage（使用者自行輸入）

**localStorage 錯誤處理：**
- 嘗試寫入前檢查 localStorage 是否可用
- 若 localStorage 已滿或被停用，顯示警告「無法儲存資料，請檢查瀏覽器設定」
- 資料仍可在當前工作階段中使用，但不會持久化

### 外部 API

**Google Sheets API：**
- 使用者需自行申請 API Key（從 Google Cloud Console）
- 僅支援「已發布到網頁」的公開試算表（不需 OAuth）
- 使用方式：
  1. 使用者在 Google Sheets 中選擇「檔案 > 發布到網頁」
  2. 取得試算表 ID（網址中 `/d/` 後的字串）
  3. 在應用程式中輸入試算表 ID 和 API Key
- API 端點：`https://sheets.googleapis.com/v4/spreadsheets/{spreadsheetId}/values/{range}?key={apiKey}`
- 預設讀取範圍：A 欄（第一欄），跳過第一列（標題列），使用者可自訂範圍如 `Sheet1!A2:A`
- 錯誤處理：
  - API Key 無效：顯示「API Key 無效，請檢查設定」
  - 試算表不存在或未公開：顯示「無法讀取試算表，請確認已發布為網頁」
  - 配額超限：顯示「API 呼叫次數已達上限，請稍後再試」
  - 網路錯誤：顯示「無法連線，請檢查網路」

## 使用者流程

```
1. 開啟頁面
2. 選擇主題
3. 輸入/匯入名單
4. 輸入/匯入獎項
5. 點擊「開始抽獎」
6. 觀看 3D 動畫
7. 中獎者揭曉（含動畫與音效）
8. 重複步驟 5-7 直到完成
9. 匯出結果（選用）
```

## 錯誤處理

- 名單為空時，提示「請先加入名單」
- 獎項為空且抽獎時，提示「無獎項，將只抽出名字」
- Google API Key 無效時，顯示錯誤訊息
- 網路錯誤時，提示「無法連線，請檢查網路」
- WebGL 不支援時：顯示「您的瀏覽器不支援 WebGL，將使用簡易模式」並切換到 2D 備用模式
  - 2D 備用模式：名字以水平滾動方式顯示，抽獎時快速滾動後減速停止，中獎者名字放大顯示（無需 Three.js，純 CSS 動畫）

## 效能考量

- Three.js 使用 WebGL 渲染
- 名單超過 50 人時，球體上只隨機顯示 50 個名字（避免效能問題）
  - 抽獎仍從完整名單中隨機抽取（不受顯示限制影響）
  - 顯示的名字在每次「抽獎動畫開始時」重新隨機選擇（非閒置時）
  - 使用者介面的名單列表顯示完整名單
- 動畫使用 requestAnimationFrame
- 響應式設計，支援不同螢幕尺寸
